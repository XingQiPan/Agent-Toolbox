# Web 控制台设计方向

本轮参考 MagicalBox 这类在线工具箱的工具分类、搜索、工具卡片和“打开即用”体验，然后落地最适合 Phase 0 演示和操作的版本。

参考信息：

- MagicalBox 首页描述其提供 100+ 免费在线工具。
- 工具类型覆盖视频压缩、音频转换、图片处理、PDF 操作、二维码生成、文本处理、单位转换等。
- 页面强调无需下载、在线使用和隐私保护。

## 方向 A：控制台工作台

特点：

- 左侧导航固定，主区域直接展示工具、运行器、结果和审计。
- 信息密度高，适合工程人员和 AI Agent 调试。
- 打开页面就能执行工具，不需要先经过宣传页。

采用状态：第一版已采用，第二版进一步升级为“工具箱 + 运行台”。

## 方向 B：插件资产库

特点：

- 插件作为第一层入口，每个插件展示工具、权限、版本、风险等级。
- 适合插件数量变多后的管理场景。
- Phase 0 工具较少，当前采用会显得页面空。

采用状态：后续插件数量增加后再升级。

## 方向 C：任务流水线

特点：

- 用户先创建任务，再从任务里选择工具、查看 artifact、追踪审计。
- 适合 PDF、图片、文件处理等多步骤工作流。
- 当前 Phase 0 还没有 File Artifact Service，暂不优先。

采用状态：Phase 2 后考虑。

## 当前 UI 范围

当前 Web 控制台提供：

- API 健康状态。
- 插件数量、工具数量、调用数量、最近耗时。
- 工具搜索、分类筛选和工具卡片。
- 输入 Schema 查看。
- JSON 输入编辑和 API 工具执行。
- 浏览器本地工具执行，包括 Base64、URL、文本统计、时间戳、UUID、命名格式和颜色转换。
- 执行结果查看。
- 最近审计日志查看。

## 第二版改进

已修复：

- 顶部导航可以真正切换工具箱、运行台、插件、审计。
- 分类标签可以筛选不同类型工具。
- 工具卡片点击后进入运行台。
- 本地工具即使 API 不在线也能使用。
- 视觉风格改为浅色在线工具站风格，弱化工程后台感。
