# Web 控制台设计方向

本轮先做三个方向判断，然后落地最适合 Phase 0 演示和操作的版本。

## 方向 A：控制台工作台

特点：

- 左侧导航固定，主区域直接展示工具、运行器、结果和审计。
- 信息密度高，适合工程人员和 AI Agent 调试。
- 打开页面就能执行工具，不需要先经过宣传页。

采用状态：已采用。

## 方向 B：插件资产库

特点：

- 插件作为第一层入口，每个插件展示工具、权限、版本、风险等级。
- 适合插件数量变多后的管理场景。
- Phase 0 工具较少，当前采用会显得页面空。

采用状态：后续插件数量增加后再升级。

## 方向 C：任务流水线

特点：

- 用户先创建任务，再从任务里选择工具、查看 artifact、追踪审计。
- 适合 PDF、图片、文件处理等多步骤工作流。
- 当前 Phase 0 还没有 File Artifact Service，暂不优先。

采用状态：Phase 2 后考虑。

## 当前 UI 范围

当前 Web 控制台提供：

- API 健康状态。
- 插件数量、工具数量、调用数量、最近耗时。
- 工具搜索和选择。
- 输入 Schema 查看。
- JSON 输入编辑和工具执行。
- 执行结果查看。
- 最近审计日志查看。
