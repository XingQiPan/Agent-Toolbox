# Web 操作界面设计方向

本轮参考 MagicalBox 这类在线工具箱的工具分类、搜索、工具卡片和“打开即用”体验，然后落地最适合 Phase 0 演示和操作的版本。

参考信息：

- MagicalBox 首页描述其提供 100+ 免费在线工具。
- 工具类型覆盖视频压缩、音频转换、图片处理、PDF 操作、二维码生成、文本处理、单位转换等。
- 页面强调无需下载、在线使用和隐私保护。
- 本轮重点参考了 `image-compress` 的上传、压缩、预览、下载操作路径，以及 `regex-collection` 的正则搜索、复制、测试路径。

## 方向 A：控制台工作台

特点：

- 左侧导航固定，主区域直接展示工具、运行器、结果和审计。
- 信息密度高，适合工程人员和智能体调试。
- 打开页面就能执行工具，不需要先经过宣传页。

采用状态：第一版已采用，第二版发现仍偏后台控制台，第三版改为“在线工具站 + 具体工具页”，第四版根据参考截图改为“顶部横栏 + 左侧分类 + 灰底工具卡网格”。

## 方向 B：插件资产库

特点：

- 插件作为第一层入口，每个插件展示工具、权限、版本、风险等级。
- 适合插件数量变多后的管理场景。
- Phase 0 工具较少，当前采用会显得页面空。

采用状态：后续插件数量增加后再升级。

## 方向 C：任务流水线

特点：

- 用户先创建任务，再从任务里选择工具、查看 artifact、追踪审计。
- 适合 PDF、图片、文件处理等多步骤工作流。
- 当前 Phase 0 还没有 File Artifact Service，暂不优先。

采用状态：Phase 2 后考虑。

## 当前 UI 范围

当前 Web 控制台提供：

- 接口健康状态。
- 顶部横栏：品牌、功能列表、全局搜索、刷新和工具运行状态。
- 左侧应用分类：日常、查询、文档、智能、图片、音频、视频、文字、加密、单位、生活。
- 首页灰底工具卡网格：可用工具可点击，规划中工具弱化展示。
- 智能体接入页：展示推荐接入流程、当前可用接口、请求/返回示例和规划中能力。
- 图片压缩页：上传、质量调节、最大宽高、格式选择、压缩前后预览、下载。
- 正则大全页：正则搜索、分类选择、编辑/复制表达式、编辑 flags、测试文本、匹配列表。
- 数据工具页：用户输入内容、接口工具执行、可读结果卡片、可复制文本和折叠的原始数据。
- 审计页：接口工具调用和浏览器本地工具调用记录。

## 第二版改进

已修复：

- 顶部导航可以真正切换工具箱、运行台、插件、审计。
- 分类标签可以筛选不同类型工具。
- 工具卡片点击后进入运行台。
- 本地工具即使接口不在线也能使用。
- 视觉风格改为浅色在线工具站风格，弱化工程后台感。

## 第三版改进

已修复：

- 将页面入口改为首页、图片压缩、正则大全、数据工具、审计。
- 顶部导航会同步更新地址栏，支持刷新、回退和直接访问工具页。
- 图片压缩不再只是展示入口，而是浏览器本地真实压缩工具。
- 正则大全不再只是展示入口，而是可搜索、编辑、复制和在线测试的工具。
- 视觉进一步贴近在线工具站，减少后台面板和工程调试感。

## 第四版改进

已修复：

- 按用户参考截图调整为上方横栏、左侧分类导航、中间工具卡网格。
- 右上角去掉登录按钮，改为当前项目需要的工具运行状态和插件/工具数量。
- 首页去掉大 Hero 和渐变背景，改为高留白、低噪音的工具集合站风格。
- 工具卡使用浅灰背景、黑白图标、标题和简短描述，信息密度更接近参考站。
- 左侧分类点击后展示对应应用分组，顶部搜索回到首页并筛选工具。
- 图片压缩、正则大全、数据工具和审计仍保留真实可操作页面。

## 第五版改进

已修复：

- Web 页面默认面向用户，界面文案使用中文。
- 数据工具输入改为用户可理解的内容输入框，不要求用户填写接口参数 JSON。
- 工具运行结果改为结果卡片、验证状态、可复制文本和指标信息。
- 原始结构化返回仅折叠展示，定位为给智能体、脚本或调试使用。

## 第六版改进

已修复：

- 后端新增 `/v1/ai/interfaces`，返回智能体可接入接口清单。
- Web 新增“智能体接入”页面，展示搜索工具、获取详情、执行工具三步流程。
- 接入页区分当前可用接口和规划中能力，避免把文件、MCP 等后续项误展示为已完成。
